# 향후 계획

## 2단계: Drive + Sheets + Analytics

### 목표

파일 관리와 데이터 분석 기능을 추가하여 업무 생산성을 높인다.

### Google Drive 도구

- `search_files` - 파일/폴더 검색
- `upload_file` - 파일 업로드
- `download_file` - 파일 다운로드
- `create_folder` - 폴더 생성
- `share_file` - 파일 공유 설정
- `list_recent` - 최근 파일 목록

### Google Sheets 도구

- `read_sheet` - 시트 데이터 읽기
- `write_sheet` - 시트 데이터 쓰기
- `append_row` - 행 추가
- `create_spreadsheet` - 새 스프레드시트 생성

### Google Analytics 도구

- `get_report` - 분석 리포트 조회
- `get_realtime` - 실시간 데이터 조회

### 크로스 서비스 연동

- 일정에 Drive 파일 첨부
- 이메일에서 첨부파일 Drive에 저장
- Sheets 데이터로 이메일 자동 생성

## 3단계: 슬랙/텔레그램 채팅 인터페이스

### 목표

Claude Code 없이도 모바일/데스크톱에서 자연어로 Jarvis를 사용한다.

### 아키텍처

```
사용자 (텔레그램/슬랙)
    │
    ▼
Cloudflare Workers (Webhook 수신)
    │
    ▼
Claude API (자연어 처리)
    │
    ▼
Jarvis MCP Server (도구 실행)
```

### 텔레그램 봇

- BotFather로 봇 생성
- Webhook 기반 메시지 수신
- Cloudflare Workers에서 처리

### 슬랙 앱

- Slack App 생성
- Slash command + 멘션 지원
- Cloudflare Workers에서 처리

### 비용 예상

- Cloudflare Workers: 무료 (일 100,000 요청)
- Claude API: ~$5/월 (예상, Haiku 모델 사용)

## 4단계: Cron 스케줄 자동화

### 목표

반복적인 작업을 자동으로 실행한다.

### 예정 기능

| 자동화 | 스케줄 | 설명 |
|--------|--------|------|
| 아침 브리핑 | 매일 08:00 | 오늘 일정 + 안 읽은 메일 요약 |
| 주간 리포트 | 매주 금요일 17:00 | 이번 주 일정/메일 요약 |
| 중요 메일 알림 | 5분마다 | 특정 발신자 메일 즉시 알림 |
| 일정 리마인더 | 이벤트 30분 전 | 곧 시작할 일정 알림 |

### 기술

- Cloudflare Workers Cron Triggers
- KV Storage로 상태 관리
- 텔레그램/슬랙으로 알림 전송

## 5단계: 자율 개발 루프

### 목표

Jarvis가 사용 패턴을 분석하고 스스로 기능을 개선한다.

### 개념

```
사용 패턴 수집 → 패턴 분석 → 개선점 도출 → 코드 생성 → 테스트 → 승인 요청 → 배포
```

### 예상 시나리오

1. **반복 패턴 감지**: "매주 월요일마다 같은 메일을 보내고 있네요"
2. **자동화 제안**: "이걸 자동으로 보내드릴까요?"
3. **승인 후 구현**: 사용자 승인 → 자동 코드 생성 → 테스트 → 배포
4. **피드백 루프**: 사용자 반응으로 개선

### 안전장치

- 모든 자동 변경에 사용자 승인 필수
- 변경 사항 롤백 기능
- 비용 상한선 설정
- 위험한 작업 (삭제, 발송) 이중 확인
